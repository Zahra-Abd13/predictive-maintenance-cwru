## 📋 Overview

Rotating machinery — motors, pumps, compressors — rely on bearings to function. Bearing failures cause unplanned industrial downtime that costs thousands of dollars per hour. Traditional maintenance is either **reactive** (fix it after it breaks) or **time-based** (replace every N hours regardless of condition). Both are wasteful.

This project builds a **Predictive Maintenance system** that detects bearing faults early using vibration sensor data and classifies the bearing's health condition into one of four states — before failure occurs.

**Phase 1** (this repository) covers the complete software ML pipeline.  
**Phase 2** (in development) will deploy the model on an ESP32 microcontroller with an ADXL355 accelerometer for real-time edge inference.

## 🎯 Results

| Model | Test Accuracy | Weighted F1 | CV F1 (5-fold) |
|---|---|---|---|
| **Random Forest** | **100.00%** | **1.0000** | **0.9977 ± 0.0014** |
| XGBoost | 99.85% | 0.9985 | 0.9970 ± 0.0015 |
| Logistic Regression | 99.70% | 0.9970 | 0.9940 ± 0.0032 |

> **Honest note:** The train/test split was performed at window level with 50% overlap, which inflates test scores slightly. The 5-fold cross-validation score of **F1 = 0.9977** is the more conservative and reliable estimate. Published benchmarks on the same CWRU dataset report F1 = 0.96–0.99.

## 📊 Dataset

**CWRU Bearing Dataset** — Case Western Reserve University Bearing Data Center  
🔗 https://engineering.case.edu/bearingdatacenter

The most widely cited benchmark dataset for bearing fault detection, used in 3,000+ research papers.

| Property | Value |
|---|---|
| Sensor | PCB Piezotronics accelerometer |
| Sampling rate | 12,000 Hz |
| Load condition | 1 HP (1772 RPM) |
| Fault introduction | Electro-discharge machining (EDM) |
| Fault diameters | 0.007", 0.014", 0.021", 0.028" |

### Classes

| Label | Class | Fault Location |
|---|---|---|
| 0 | Normal | No fault |
| 1 | Inner Race | Inner raceway |
| 2 | Ball | Rolling element |
| 3 | Outer Race | Outer raceway |

### Files Used (11 files, 1HP load)

`normal_1`, `IR_007_1`, `IR_014_1`, `IR_021_1`, `IR_028_1`,  
`ball_007_1`, `ball_014_1`, `ball_021_1`, `ball_028_1`,  
`OR_007_1`, `OR_021_1`

> `OR_014_1.mat` was excluded — EDA revealed its RMS and kurtosis were statistically identical to Normal, making it harmful for training.

## ⚙️ Pipeline

```
Raw .mat Files  (12,000 Hz vibration signal)
      │
      ▼
01_eda.ipynb
  • Load and visualize raw signals
  • FFT spectrum analysis per class
  • Statistical summary — identify anomalous files
      │
      ▼
02_preprocessing.ipynb
  • DC offset removal
  • Butterworth low-pass filter (5000 Hz cutoff, order 5)
  • Signal windowing: 1024 samples, 50% overlap (~85ms per window)
  • Stratified train/test split (80/20)
  • Saves: X_windows.npy, y_labels.npy, X_train.npy, X_test.npy
      │
      ▼
03_features.ipynb
  • Extract 16 features per window (9 time-domain + 7 frequency-domain)
  • StandardScaler fitted on training data only
  • SMOTE applied after scaling on training data only
  • ANOVA F-score feature ranking
  • Saves: F_train_smote.npy, F_test_scaled.npy, scaler.pkl, features.csv
      │
      ▼
04_modeling.ipynb
  • Train: Logistic Regression, Random Forest, XGBoost
  • 5-fold cross-validation
  • Confusion matrices, ROC curves, classification report
  • SHAP global feature importance
  • RandomizedSearchCV hyperparameter tuning
  • SelectFromModel feature selection
  • Saves: inference_pipeline.pkl, model_results.csv
```

## 🔬 Feature Engineering

### Time-Domain Features (9)

| Feature | Formula | Physical Meaning |
|---|---|---|
| RMS | √(Σxᵢ²/N) | Overall vibration energy |
| Kurtosis | μ₄/σ⁴ | Impulsive spikes from bearing impacts |
| Crest Factor | peak/RMS | Sharpness of impulse peaks |
| Skewness | μ₃/σ³ | Asymmetry of amplitude distribution |
| Peak | max\|x\| | Maximum absolute amplitude |
| Peak-to-Peak | max−min | Total swing range |
| Std | σ | Spread of amplitudes |
| Mean Abs | Σ\|x\|/N | Average rectified signal level |
| Shape Factor | RMS/mean_abs | Waveform shape index |

### Frequency-Domain Features (7)

| Feature | Physical Meaning |
|---|---|
| Dominant Frequency | Frequency with highest FFT magnitude |
| Spectral Energy | Total power in frequency domain |
| Spectral Entropy | Spread of energy — faults concentrate at specific frequencies |
| Spectral Mean | Average FFT magnitude |
| Spectral Std | Variation in FFT magnitudes |
| Band Energy 100–4000 Hz | Energy in fault-characteristic frequency band |
| Band Energy 4000–6000 Hz | Energy in noise band |

> A **Hanning window** is applied before FFT to reduce spectral leakage. The fault energy band was updated from 3000 → 4000 Hz after EDA showed fault peaks between 2500–4000 Hz.

---

## 🧠 Key Engineering Decisions

| Decision | Reason |
|---|---|
| Excluded `OR_014_1.mat` | RMS=0.093, kurtosis=-0.06 — identical to Normal. Would teach wrong labels |
| Removed signal normalization | Normal RMS=0.066 vs Outer Race RMS=0.59 (9× difference). Amplitude energy is discriminative |
| Band energy 100→4000 Hz | FFT analysis showed fault peaks at 2500–4000 Hz, above original 3000 Hz cutoff |
| Hanning window on FFT | Reduces spectral leakage — fault peaks become sharper and more discriminative |
| Split before feature extraction | Prevents data leakage through StandardScaler and SMOTE |
| SMOTE after scaling | Interpolation in standardized feature space is geometrically correct |
| Kept all 16 features | Tree models find non-linear combinations. ANOVA is only linear. SHAP is the real arbiter |

---

## 📈 SHAP Feature Importance

Top features by SHAP (Random Forest, global):

1. `spectral_mean` — average FFT magnitude scales with fault severity
2. `shape_factor` — waveform changes from sinusoidal (normal) to impulsive (faulty)
3. `peak_to_peak` — total amplitude range increases with fault size
4. `peak` — maximum spike amplitude
5. `rms` — overall energy level
6. `band_energy_100_4000` — fault energy concentration

> Note: SHAP ranking differs from ANOVA ranking because SHAP captures non-linear feature interactions inside tree nodes, while ANOVA measures only linear separability.

## ⚠️ Limitations

- **Window-level splitting:** overlapping windows from the same file appear in both train and test sets. A file-level holdout would give a more conservative accuracy estimate.
- **Single operating condition:** all training data from 1HP load. Validation on 0HP files produced 0% accuracy — the model does not currently generalize across load conditions. Phase 2 fix: multi-load training data.
- **Single bearing type:** CWRU bearing dimensions determine fault characteristic frequencies. Different bearing models will have different frequencies.
- **Sensor dependency:** trained on 12,000 Hz data. Deployment on lower-rate sensors (e.g. MPU6050 at ~1000 Hz) loses the fault frequency band.

## 🚀 Installation & Usage

### Prerequisites

- Python 3.10+
- Anaconda (recommended)

### Setup

```bash
# Clone the repository
git clone https://github.com/Zahra-Abd13/predictive-maintenance-cwru.git
cd predictive-maintenance-cwru

# Install dependencies
pip install -r requirements.txt

# Download CWRU data files
# See data/raw/README.md for download instructions
```

### Run Notebooks

Open Jupyter and run notebooks in order:

```bash
jupyter notebook
```

1. `01_eda.ipynb` — explore raw signals
2. `02_preprocessing.ipynb` — filter, window, split
3. `03_features.ipynb` — extract features, SMOTE, scale
4. `04_modeling.ipynb` — train, evaluate, SHAP

### Run the Web App

```bash
streamlit run app.py
```

Upload any CWRU `.mat` file to get a real-time fault classification.
